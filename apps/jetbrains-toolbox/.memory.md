## jetbrains-toolbox

JetBrains Toolbox App packaged in a Distrobox container. Toolbox manages JetBrains IDEs (IDEA, PyCharm, GoLand, etc.) installed into the shared `$HOME` — so IDE binaries and their `.desktop` files appear on the host immediately without any extra export step.

## Implementation

- Base: `docker.io/ubuntu:24.04` (officially supported by JetBrains; 22.04 and 24.04 LTS listed)
- Export: `gui:jetbrains-toolbox:JetBrains Toolbox` — system-tray app, `Terminal=false`
- Toolbox binary downloaded at image-build time via JetBrains release API (`code=TBA`) to `/opt/jetbrains-toolbox/`; symlinked to `/usr/bin/jetbrains-toolbox`
- SDKMAN installed system-wide to `/opt/sdkman` with default Java LTS pre-installed; sourced from `/etc/bash.bashrc` and `/etc/zsh/zshrc` so it's available in both shells
- Developer tools included: `zsh`, `git`, `git-lfs`, `build-essential`, `cmake`, `python3`, `ripgrep`, `shellcheck`, `gh` (GitHub CLI), `docker-ce-cli` (Docker client only)

On first run, Toolbox creates `~/.local/share/applications/jetbrains-toolbox.desktop` in the shared `$HOME`, overwriting the distrobox-exported one. This is fine — the Toolbox-managed `.desktop` is better.

IDEs installed by Toolbox go to `~/.local/share/JetBrains/Toolbox/apps/` (shared `$HOME`) and are launched from the host directly. JetBrains bundles its own JDK so they don't need the container environment.

## Pitfalls and notes

- **Docker CLI socket**: no active Podman socket found on this host at time of setup. Docker CLI is installed but needs `podman system service --time=0 &` running on the host to get a socket at `$XDG_RUNTIME_DIR/podman/podman.sock`. If needed, add `-v $XDG_RUNTIME_DIR/podman/podman.sock:/var/run/docker.sock` to `create_flags` and export `DOCKER_HOST`.
- **SDKMAN version pinning**: `sdk install java` installs the current SDKMAN default at image build time (typically latest Temurin LTS). Re-building the image later may pull a newer Java version.
- **Toolbox `.desktop` overwrite**: Toolbox replaces the distrobox-exported `.desktop` on first run. Expected behaviour — do not re-export after initial setup unless Toolbox is reinstalled.
- **System tray icon shows white exclamation mark**: The tray PNG (`/opt/jetbrains-toolbox/bin/toolbox-tray-color.png`) lives inside the container; the host tray can't reach that path and falls back to a generic icon. Fixed via a wrapper script at `/usr/bin/jetbrains-toolbox` that copies it to `~/.local/share/icons/hicolor/256x256/apps/jetbrains-toolbox.png` (shared `$HOME`) on first launch.
- **IDEs outside the container**: Toolbox-generated `.desktop` files for individual IDEs invoke the IDE binary directly on the host (from `$HOME`). They run outside the container environment. This is fine because JetBrains IDEs ship a bundled JDK and most UI libs are present on any modern desktop.
