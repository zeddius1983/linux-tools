## opencode

OpenCode is an open-source AI coding agent with a terminal TUI. Packaged as a Distrobox container so it can be exported to the host shell and app menu without polluting the host environment.

## Implementation

- **Base image**: `docker.io/ubuntu:24.04` — minimal, matches other CLI tool apps (codex-cli pattern)
- **Install method**: official install script `https://opencode.ai/install | sh`
  - Script installs to `$HOME/.opencode/bin/opencode`; during image build `$HOME=/root`, so the binary lands at `/root/.opencode/bin/opencode`
  - Copied to `/usr/bin/opencode` so `command -pv` (system PATH) finds it for distrobox-export
- **Exports**:
  - `bin:opencode` — exports binary to `~/.local/bin` on the host for shell use
  - `desktop:opencode:OpenCode` — creates a terminal-launched `.desktop` entry in the app menu
- **No extra flags**: pure TUI, no GPU/device passthrough needed

## Pitfalls and notes

- `/usr/local/bin` is not in the system PATH used by `command -pv`; the binary must be at `/usr/bin/opencode` for exports to work
- The install script sets `INSTALL_DIR=$HOME/.opencode/bin` — if this changes in a future version, the `cp` path in the Dockerfile will need updating
- The install script uses `set -euo pipefail` and requires bash. Pipe to `bash`, not `sh` — Ubuntu's `sh` is `dash` and will fail with "Illegal option -o pipefail"
- API keys are configured by the user at runtime (not baked into the image)
