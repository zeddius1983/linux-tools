## comfyui

Node-based UI for generative AI image and video workflows (Stable Diffusion, SDXL, Flux, WAN video, etc.). Packaged as a Distrobox container with AMD ROCm GPU acceleration, optimised for Strix Halo (gfx1151) unified memory.

## Implementation

**Base image**: `rocm/pytorch:rocm7.2.4_ubuntu24.04_py3.12_pytorch_release_2.10.0`
- Official AMD image; PyTorch + ROCm 7.2.4 pre-installed; no separate ROCm installation needed.
- Supports gfx1151 (Strix Halo) — earlier nightlies-only workaround (kyuz0 repo) is no longer needed as of ROCm 7.1+.

**Install path**: `/opt/ComfyUI` (cloned at image-build time with `--depth=1`)

**Python**: `/opt/venv/bin/python` — the rocm/pytorch image installs everything into `/opt/venv`. The system Python at `/usr/bin/python` has no site-packages. The wrapper must use the full venv path explicitly.

**Model storage**: `~/comfy-models/` — configured via `/opt/ComfyUI/extra_model_paths.yaml`. Lives in shared `$HOME` so models survive container rebuilds. Sub-dirs: checkpoints, vae, loras, controlnet, upscale_models, clip, unet, diffusion_models.

**Plugins**: ComfyUI-Manager pre-installed at `custom_nodes/ComfyUI-Manager`; users install further nodes from the UI.

**Export type**: `desktop:comfyui:ComfyUI` (opens terminal with server logs) + `bin:comfyui` (PATH entry). ComfyUI is a web server; access via browser at http://localhost:8188.

**create_flags**: `--device /dev/kfd --device /dev/dri --group-add video --group-add render --security-opt seccomp=unconfined --ipc=host`
- `/dev/kfd` is the AMD ROCm compute device; NOT auto-passed by Distrobox.
- `--security-opt seccomp=unconfined` required for some ROCm HIP operations.
- `--ipc=host` needed for ROCm shared-memory IPC.

## Working state

Confirmed working as of 2026-06-05 with `rocm/pytorch:rocm7.2.4_ubuntu24.04_py3.12_pytorch_release_2.10.0`. ComfyUI 0.24.0, ROCm 7.2, gfx1151 detected, NORMAL_VRAM state with 131 GB unified memory.

## Pitfalls and notes

- **`/opt` permissions**: `/opt/ComfyUI` is installed by root during the image build. ComfyUI creates `user/`, `output/`, `temp/` at runtime and fails with `PermissionError` if the Distrobox user can't write there. Fix: `RUN chmod -R a+rwX /opt/ComfyUI` in the Dockerfile.
- **`rocm/pytorch` venv**: all packages are in `/opt/venv`, not the system Python. The wrapper must use `/opt/venv/bin/python` explicitly — bare `python` resolves to `/usr/bin/python` which has no site-packages.
- **`--group-add render/video` on Ubuntu**: use numeric GIDs (`--group-add 992 --group-add 44`) — Ubuntu images don't have these groups in `/etc/group` and Podman fails to start the container if they're missing.
- **`--disable-mmap` is critical for Strix Halo**: memory mapping above 64 GB is very slow on gfx1151 due to a ROCm/driver bug (documented as of early 2026). Causes hangs during model load if omitted. May be fixed in future ROCm releases.
- **`--bf16-vae`**: prevents OOM during VAE decode with Strix Halo unified memory.
- **`--cache-none`**: disables model caching; helps manage GTT vs RAM allocation on large unified memory systems.
- **`--reserve-vram 3`**: recommended for 16 GB+ VRAM (reserves 3 GB for system + PyTorch overhead).
- **`TORCH_ROCM_AOTRITON_ENABLE_EXPERIMENTAL=1`**: enables AOTriton kernels for gfx1151 — required for good performance.
- **`TORCH_BLAS_PREFER_HIPBLASLT=1`**: enables recent hipBLASLt improvements for gfx1151 (ROCm/rocm-libraries#3913, #3879).
- **`HSA_ENABLE_SDMA=0`**: disables SDMA engine; improves stability on AMD consumer GPUs.
- **MIOpen first-run slowness**: first generation takes ~2× longer while MIOpen compiles GPU kernels. Cache persists at `~/.cache/miopen` across runs.
- **base image size**: `rocm/pytorch` images are 15–20 GB. Build + pull time is significant.
- **rocm/pytorch image version**: use the latest available tag from Docker Hub (`rocm/pytorch`). Check for newer tags if GPU perf is unexpectedly poor.
- **pip install during build**: `pip install -r requirements.txt` skips torch/torchvision/torchaudio because they are already satisfied by the base image version.
