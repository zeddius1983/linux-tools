## unsloth

Fast LLM fine-tuning library with AMD ROCm GPU acceleration. Packaged as a Distrobox container exposing `unsloth` (Python runner for training scripts) and `unsloth-studio` (web UI on port 8888).

## Implementation

**Base image**: `docker.io/rocm/dev-ubuntu-24.04:7.2.4-complete` — same as llama-cpp, provides the full ROCm 7.2.4 toolchain.

**Python venv at `/opt/unsloth`** (Python 3.11). Packages installed in order:
1. `uv` (fast pip replacement)
2. PyTorch + torchvision + torchaudio via `https://download.pytorch.org/whl/rocm7.2` — rocm7.2 wheels start at torch 2.11.0 (unlike rocm7.1 which is capped below 2.11.0 in Unsloth docs).
3. `unsloth[amd]` via uv
4. ROCm bitsandbytes from GitHub release URL (not PyPI — PyPI release has no ROCm support).

**Exported commands** (both in `/usr/bin/` for `command -pv`):
- `unsloth` — calls `/opt/unsloth/bin/python "$@"` with AMD env vars set.
- `unsloth-studio` — calls `/opt/unsloth/bin/unsloth studio -H 0.0.0.0 -p 8888`.

**AMD env vars** set in both wrappers: `HF_HUB_DISABLE_XET=1`, `PYTORCH_HIP_ALLOC_CONF=expandable_segments:True`, `HSA_ENABLE_SDMA=0`.

**`create_flags`**: identical to vllm — `--device /dev/kfd --device /dev/dri --group-add 44 --group-add 992 --cap-add SYS_PTRACE --security-opt seccomp=unconfined --ipc=host`.

**HF_TOKEN**: not baked in — inherited from host shell at runtime.

**Model cache**: `~/.cache/huggingface/` via Distrobox shared `$HOME`.

## Pitfalls and notes

- **PyTorch rocm7.2 vs. Unsloth docs**: Unsloth's AMD install docs pin `torch<2.11.0` with `rocm7.1`. The rocm7.2 wheel index starts at 2.11.0 and 2.12.0. If a future `unsloth[amd]` release enforces `torch<2.11.0` as a hard dep, the build will fail — fallback is to switch the index to `rocm7.1` and reinstate the upper cap.
- **ROCm bitsandbytes URL**: bitsandbytes on PyPI lacks ROCm support. Must install from `https://github.com/bitsandbytes-foundation/bitsandbytes/releases/download/continuous-release_main/bitsandbytes-1.33.7.preview-py3-none-manylinux_2_24_x86_64.whl`. URL contains the version — check for newer versions when updating.
- **Unsloth Studio AMD training**: As of packaging (2026-06-11), Studio's training feature does not support AMD GPU — only NVIDIA and Intel are supported. Chat and Data Recipes work on AMD. Tracked as "coming soon" upstream.
- **`SYS_PTRACE`**: Required by PyTorch internals (same reason as vllm). Without it, some PyTorch operations that use ptrace for profiling may fail at runtime.
- **`--ipc=host`**: Required for PyTorch shared-memory multiprocessing (DataLoader workers).
- **`/usr/bin/` wrapper placement**: `command -pv` (used by tools.sh export logic) does not search `/usr/local/bin`. Both wrappers are placed in `/usr/bin/`.
