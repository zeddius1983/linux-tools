## unsloth

Fast LLM fine-tuning library with AMD ROCm GPU acceleration. Packaged as a Distrobox container exposing `unsloth` (Python runner for training scripts) and `unsloth-studio` (web UI on port 8888).

## Implementation

**Base image**: `docker.io/rocm/pytorch:rocm7.2.4_ubuntu24.04_py3.12_pytorch_release_2.10.0` — same as comfyui. PyTorch 2.10.0 + ROCm 7.2.4 + Python 3.12 venv at `/opt/venv` are all pre-installed; no separate Python or PyTorch installation needed.

**Python venv**: `/opt/venv` (from base image, Python 3.12 + PyTorch 2.10.0 ROCm). Packages installed on top:
1. `uv` (fast pip replacement)
2. `unsloth[amd]` via uv
3. ROCm bitsandbytes from GitHub release URL (not PyPI — PyPI release has no ROCm support)

**Studio home layout**: `unsloth_cli` expects `STUDIO_HOME/unsloth_studio/bin/python` + `share/studio.conf` sentinel. Since `$HOME` is the shared host home at runtime (not known at build time), the `unsloth-studio` wrapper creates `~/.unsloth/studio/unsloth_studio` as a symlink to `/opt/venv` on first run.

**Exported commands** (both in `/usr/bin/` for `command -pv`):
- `unsloth` — calls `/opt/venv/bin/python "$@"` with AMD env vars set.
- `unsloth-studio` — sets up `~/.unsloth/studio/` on first run, then calls `/opt/venv/bin/unsloth studio -H 0.0.0.0 -p 8888`.

**AMD env vars** set in both wrappers: `HF_HUB_DISABLE_XET=1`, `PYTORCH_HIP_ALLOC_CONF=expandable_segments:True`, `HSA_ENABLE_SDMA=0`.

**`create_flags`**: identical to vllm — `--device /dev/kfd --device /dev/dri --group-add 44 --group-add 992 --cap-add SYS_PTRACE --security-opt seccomp=unconfined --ipc=host`.

**HF_TOKEN**: not baked in — inherited from host shell at runtime.

**Model cache**: `~/.cache/huggingface/` via Distrobox shared `$HOME`.

## Pitfalls and notes

- **Use `rocm/pytorch` base, not `rocm/dev-ubuntu`**: The `rocm/dev-ubuntu-24.04` base doesn't ship Python, requiring a manual apt install (which broke when python3.11 was requested on Noble). The `rocm/pytorch` base gives Python 3.12 + PyTorch pre-installed in `/opt/venv`.
- **Official one-line installer doesn't work at image build time**: it ignores `HOME=` inline env overrides in `curl | sh` pipelines and creates its own Python 3.13 venv from scratch, ignoring the base image's pre-installed PyTorch.
- **Studio home can't be set at build time**: `$HOME` during image build is `/root`, which is inaccessible to the Distrobox user at runtime. The `unsloth-studio` wrapper creates `~/.unsloth/studio/unsloth_studio → /opt/venv` on first run instead.
- **ROCm bitsandbytes URL**: bitsandbytes on PyPI lacks ROCm support. Must install from `https://github.com/bitsandbytes-foundation/bitsandbytes/releases/download/continuous-release_main/bitsandbytes-1.33.7.preview-py3-none-manylinux_2_24_x86_64.whl`. URL contains the version — check for newer versions when updating.
- **Unsloth Studio AMD training**: As of packaging (2026-06-11), Studio's training feature does not support AMD GPU — only NVIDIA and Intel are supported. Chat and Data Recipes work on AMD. Tracked as "coming soon" upstream.
- **`SYS_PTRACE`**: Required by PyTorch internals (same reason as vllm). Without it, some PyTorch operations that use ptrace for profiling may fail at runtime.
- **`--ipc=host`**: Required for PyTorch shared-memory multiprocessing (DataLoader workers).
- **`/usr/bin/` wrapper placement**: `command -pv` (used by tools.sh export logic) does not search `/usr/local/bin`. Both wrappers are placed in `/usr/bin/`.
