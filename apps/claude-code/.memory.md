## claude-code

Packages the Claude Code CLI (Anthropic's AI coding assistant) into a Distrobox container and exports it to the host as both a binary (`claude`) and a terminal desktop shortcut.

## Implementation

- Base image: `ubuntu:24.04`
- Claude Code installed via the **official native-binary install script** (`curl -fsSL https://claude.ai/install.sh | bash`)
  - Run with `HOME=/opt/claude-install` so the installer's outputs land in a build-time directory
  - The script drops a symlink at `$HOME/.local/bin/claude` pointing to `$HOME/.local/share/claude/versions/<X.Y.Z>` (the real ~240 MB native binary)
  - We `cp -L` the dereferenced binary to `/usr/bin/claude` (a system path Distrobox's home-overlay does not hide) and delete `/opt/claude-install`
- GitHub CLI (`gh`) installed via the official GitHub APT repo
- Exports: `bin:claude` (host PATH) and `desktop:claude:Claude Code` (terminal shortcut)

## Pitfalls and notes

- **Why the install script and not APT?** The APT repo (`https://downloads.claude.ai/claude-code/apt/stable`) lags the install script by many patch versions (observed 2.1.153 on APT vs 2.1.178 from the script on the same day). The install script tracks `latest` from the release manifest.
- **Why override `$HOME` during the install?** The installer writes the launcher to `$HOME/.local/bin/claude`. Without override, image-build `$HOME=/root` puts it in `/root/.local/bin/`, which Distrobox's home-overlay hides at runtime → `cannot find 'claude' inside container`. Build-time `HOME=/opt/claude-install` keeps the install path off of any home path, then we copy the actual binary to `/usr/bin/`.
- **Why `cp -L`, not `mv`?** The installer's `$HOME/.local/bin/claude` is a symlink to `$HOME/.local/share/claude/versions/<X.Y.Z>`. A plain `mv` of the symlink would dangle once we remove `/opt/claude-install`. `cp -L` dereferences the symlink and produces a standalone binary that runs without the original layout (verified by deleting `/opt/claude-install` and running `/usr/bin/claude --version`).
- **No `nodejs`/`npm` needed.** Claude Code ships as a single self-contained native binary now, not an npm package.
- **No self-update at runtime.** Container `/usr/bin/claude` is read-only to the user. The container's `claude install latest` (if invoked) would write to `$HOME/.local/...` on the host, which would *not* be picked up by the host-exported `bin:claude` wrapper (it always invokes the container's `/usr/bin/claude`). To upgrade, rebuild the image: `tools setup claude-code`.
- **The install script also writes `.claude.json` and `.claude/backups/` under `$HOME`.** These end up in `/opt/claude-install` and are discarded with the cleanup; user config at runtime lives on the host's `~/.claude/` as expected.
