## claude-code

Packages the Claude Code CLI (Anthropic's AI coding assistant) into a Distrobox container and exports it to the host as both a binary (`claude`) and a terminal desktop shortcut.

## Implementation

- Base image: `ubuntu:24.04`
- Claude Code installed via the **official Anthropic APT repository** (`https://downloads.claude.ai/claude-code/apt/stable`)
- GitHub CLI (`gh`) installed via the official GitHub APT repo
- Exports: `bin:claude` (host PATH) and `desktop:claude:Claude Code` (terminal shortcut)

## Pitfalls and notes

- **Do not use the `curl -fsSL https://claude.ai/install.sh | bash` install script in the Dockerfile.** The script installs `claude` into `~/.local/bin/`, which at image-build time is `/root/.local/bin/`. At runtime, Distrobox overlays the container's home with the host's `$HOME`, hiding everything the image placed there. The result is `cannot find 'claude' inside container` during export. The APT method installs to `/usr/bin/claude` (a system path unaffected by the home overlay) and must be used instead.
- `nodejs` and `npm` do not need to be installed separately — the APT package pulls in its own Node.js dependency.
