## zsh-box

Host-only installer for Zsh and a curated set of modern shell utilities. Unlike all other apps in this repo, the Distrobox container is used purely as a **portable download/install environment** — nothing runs inside the container at runtime. All binaries end up in `~/.local/bin`, configs in `~/.config/zsh/conf.d/`, and zsh plugins in `~/.local/share/zsh/plugins/`. The container persists so setup can be re-run for updates.

## Implementation

- **Base image**: `ubuntu:24.04` — needs only curl, git, jq, unzip, tar.
- **Export**: `bin:zsh-install` only. The script is `COPY`-ed into the image at `/usr/bin/zsh-install` (not written inline via `RUN printf` because it's ~250 lines).
- **`host-only` marker**: empty `apps/zsh-box/host-only` file. `lib/tui.sh` and `lib/commands.sh` check for this and show dash placeholders plus a final `host` label instead of normal image/box state.
- **Wizard**: `wizard/00-tools.packages` — new `.packages` wizard type added to `lib/wizard.sh`. Handler (`_wizard_apply_packages`) passes selected tool names as a space-separated string to `distrobox enter zsh-box-box -- zsh-install --tools "..."`.
- **zsh itself**: installed on the host via `distrobox-host-exec sudo apt-get/dnf/pacman install zsh`. OS detected from `/etc/os-release`.
- **Utilities**: static musl binaries downloaded from GitHub releases API via `gh_download_latest()`. Each tool has a hardcoded asset regex pattern for the musl variant.
- **Config strategy**: `~/.zshrc` is overwritten (existing backed up as `~/.zshrc.bak.YYYYMMDD-HHMMSS`). Per-tool init snippets go to `~/.config/zsh/conf.d/NN-tool.zsh`, sourced by a glob loop in `.zshrc`.
- **Plugins**: cloned with `git clone --depth=1` to `~/.local/share/zsh/plugins/`. Sourced directly — no plugin manager.
- **Starship**: downloaded from the latest GitHub release tarball; `gruvbox-rainbow` preset written to `~/.config/starship.toml` only if the file doesn't already exist.
- **delta**: configures `git config --global` for `core.pager`, `delta.side-by-side`, etc. via the `18-delta.zsh` conf.d fragment (runs once on first shell start).

## Pitfalls and notes

- **`local` outside functions**: bash only allows `local` inside functions. Earlier draft used `local` at top-level script scope — this silently does nothing or errors depending on bash version. All variable declarations in `zsh-install` are plain assignments at script scope.
- **`gh_download_latest` regex**: the jq `test()` regex must be anchored (`^...$`) to avoid false-positive matches between e.g. `bat-v0.24.0-x86_64-unknown-linux-musl.tar.gz` and the `-gnu` variant.
- **fzf keybindings**: the key-bindings and completion shell scripts are separate from the binary and must be downloaded from the source repo at the same tag. They are saved to `~/.local/share/` and sourced in `10-fzf.zsh`.
- **atuin / zoxide / starship downloads**: use GitHub release tarballs via `gh_download_latest()` instead of `curl | bash` install scripts, which avoids executing mutable remote shell scripts during setup.
- **yazi zip layout**: the zip extracts to `yazi-<triple>/yazi` (and optionally `ya`). The full subdir path must be used when calling `install`.
- **`host-only` TUI column width**: `app_status_detail()` prints 7- and 32-column dash placeholders plus a final `host` label so the host-only row stays aligned with the normal `img_id | img_ref | box_str` layout.
- **shell change**: `chsh` requires the shell to be listed in `/etc/shells`. Installing via the host package manager handles this automatically. Static zsh builds do not register in `/etc/shells`.
